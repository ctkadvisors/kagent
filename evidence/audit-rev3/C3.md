## 1. Per-Fix Audit

### B5 (67124d9) — `MIN_SAFE_MIN=1` + `normalizeBounds`

**Status: PARTIALLY OPEN — router path bypass survives (Confidence: 92)**

The model-watch fix at `packages/llm-gateway/src/model-watch.ts:179` (`Math.max(1, ep.spec.minSafe ?? 1)`) and the PATCH validator at `packages/workbench-api/src/routes/gateway.ts:147` are both correct.

A second code path is not fixed: `packages/llm-gateway/src/model-index.ts:147` returns `minSafe: ep.spec.minSafe ?? 1` from `ModelIndex.lookup()`. For a CR with `spec.minSafe: 0`, this returns `0` (nullish coalescing does not filter the falsy value `0`). `packages/llm-gateway/src/router.ts:148-153` calls `deps.aimd.updateBounds(endpoint.model, backendUrl, { seed: lookup.seed, max: lookup.max, minSafe: lookup.minSafe })` on every request. `packages/llm-gateway/src/aimd.ts:103-105` sets `state.bounds = bounds` directly. This overwrites the model-watch's previously-clamped `bounds.minSafe: 1` back to `0` on the first request after watch-time normalization.

**Attack reproduction:** Create a CR with `spec.minSafe: 0`. Model-watch fires, clamps to 1 in AIMD state. First HTTP request arrives. `modelIndex.lookup()` returns `minSafe: 0`. Router calls `aimd.updateBounds(model, url, { minSafe: 0 })`. AIMD state now has `bounds.minSafe = 0`. Every subsequent `onError` computes `Math.max(0, floor(cap/2))` — cap can reach 0 and stays there.

A third read site at `packages/llm-gateway/src/admin-routes.ts:183` (`minSafe: ep.spec.minSafe ?? 1`) is display-only (does not write to AIMD), but shows operators a misleading `minSafe: 0` in `/admin/capacity`.

**Existing CR on upgrade path:** A CR with `spec.minSafe: 0` already in the cluster gets clamped by the watch at next informer resync, but the router undoes it on the next request. The DoS window is exactly one event cycle — effectively zero protection.

**One-line fix required at `model-index.ts:147`:**
Change `minSafe: ep.spec.minSafe ?? 1` to `minSafe: Math.max(1, ep.spec.minSafe ?? 1)`. Same fix at `admin-routes.ts:183`.

**Severity: HIGH** (restores original B5 DoS for CRs with `spec.minSafe: 0`)

**Fix:** `model-index.ts:147` — change `ep.spec.minSafe ?? 1` to `Math.max(1, ep.spec.minSafe ?? 1)`. Also apply at `admin-routes.ts:183` for display accuracy.
