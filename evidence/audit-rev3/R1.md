### 3.1 The substrate-thick compaction lane is now broadly populated

But **the substrate-thick lane is now a crowded design space**: Anthropic (configurable trigger), MAF (full Trigger/Strategy/Pipeline), Cloudflare Project Think (macro+micro), `@context-chef/ai-sdk-middleware` (community Vercel AI middleware). But the marketing-tone of "no surveyed peer ships any context-pressure primitive" needs to be retired in favor of "every surveyed peer chose substrate-thick; kagent intentionally chose substrate-thin."

A casual reader seeing "JWT-based authority" in both projects' docs will not register the technical distinction (`narrowing on spawn` vs `OBO identity`) without explicit guidance.

The technical distinction is real and defensible, but the load-bearing artifact is now `RFC-CAPABILITY-NARROWING.md` §1.2 — that section already names AgentField, Kagenti, MAF, AWS Cedar, Macaroons; **add Solo.io kagent-enterprise OBO** as a fifth adjacent-not-equivalent.
