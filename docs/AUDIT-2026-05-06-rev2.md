**The genuinely-new defensible primitive R1 identified:** the v0.1.9 context-awareness slate combines three properties simultaneously that **no surveyed competitor** ships:

1. Substrate-side, not SDK-side, refusal — `LLMClientHttpError(0, 'context_window_substrate_refused:...')` thrown by the executor before forwarding to provider.
2. Hard refusal at fixed % threshold (default 0.95) — distinct from upstream-400 reactive handling.
3. Out-of-band detector (`context_pressure_ignored`) flagging "agent ignored its context budget."

Anthropic Claude Code ships auto-COMPACT at ~95% — substantively different design (substrate owns compaction quality vs. kagent's "application owns it; substrate just enforces"). The argument in `CONTEXT-AWARENESS.md §2` is correct against the surveyed alternatives.

**R1 strategic recommendation:** drop "only OSS K3s-native" framing entirely. Lead with two narrower, evidence-backed claims:
- (a) substrate-thin context-pressure handling — no other OSS substrate ships the triple
- (b) sealed-JWT capability narrowing on spawn — no other OSS K8s-native operator ships ⊆-on-spawn enforcement

## Wired-but-dead-code paradigm — what NB1 forced into the project

The single most durable artifact from this audit cycle is [`evidence/audit-rev2/WIRED-BUT-DEAD-CODE-PARADIGM.md`](../evidence/audit-rev2/WIRED-BUT-DEAD-CODE-PARADIGM.md). NB1 was the symptom; the discipline doc names the failure mode and gives a four-class taxonomy:

- **WBD** — optional dep with fallback that fires unconditionally because the production wireup omits the dep. Tests inject; production doesn't. Bug.
- **MCALL** — required dep wired correctly, but a callsite that should invoke a method is missing. Sibling pattern; e.g. H18.
- **CSPREAD** — `...(condition && { dep })` legitimate feature flag. Not a bug.
- **DEADBRANCH** — `if (deps.X)` block reachable but body unrealized.

Every fix wave's report includes a paradigm scan in scope. Aggregate findings:
- **WBD sites confirmed and fixed:** NB1 (`tokenUtilizationSnapshot`), WBD-OP-1 (`auditEmit` in AgentWorkflow controller).
- **MCALL sites:** H18 (`apiKeyRepo.touchLastUsed` never called) — fixed in W2.
- **DEADBRANCH:** WBD-OP-2 (`listChildrenForParent` block in supervision-router.ts was a comment-only branch) — closed by W3-Operator's M2 fix.
- **CSPREAD inventory:** ~10 sites across pod (timeoutSeconds, OTel) and operator (audit-best-effort, Tenant resolver) — all confirmed legitimate.
- **Net new at HEAD:** see `WBD-FINAL-SCAN.md` for the final arbiter sweep result.
