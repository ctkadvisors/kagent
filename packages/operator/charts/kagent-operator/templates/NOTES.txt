{{- /*
v0.1.8 secret-hygiene + WS-A security baseline — deprecation warnings.

Plaintext values for sensitive fields still WORK end-to-end (the chart
renders an inline `value:` env on the operator pod, and the operator
copies it onto every spawned agent Job's env). They are DEPRECATED
because the plaintext lands in:

  - the operator Deployment's PodSpec (visible in
    `kubectl get deploy ... -o yaml` and in any GitOps audit trail)
  - every spawned agent Job's etcd object (visible in
    `kubectl get pod <agent-pod> -o yaml`)
  - the agent-pod's `/proc/<pid>/environ` at runtime

The secret-ref form keeps the plaintext on the kubelet's read path
ONLY (resolved into the running container's env at start time, never
written into etcd or any pod spec).
*/}}
